class pioran.core.GaussianProcess(function: pioran.acvf_base.CovarianceFunction | pioran.psd_base.PowerSpectralDensity, observation_indexes: jax.Array, observation_values: jax.Array, observation_errors: jax.Array | None = None, S_low: float = 10, S_high: float = 10, method: str = 'FFT', use_tinygp: bool = False, n_components: int = 0, estimate_variance: bool = True, estimate_mean: bool = True, scale_errors: bool = True, log_transform: bool = False, nb_prediction_points: int = 0, propagate_errors: bool = True, prediction_indexes: jax.Array | None = None, use_celerite: bool = False, use_legacy_celerite: bool = False)[source]#

Bases: equinox.Module

Gaussian Process Regression of univariate time series.

Model the time series as a Gaussian Process with a given covariance function or power spectral density. The covariance function is given by a CovarianceFunction object and the power spectral density is given by a PowerSpectralDensity object.

Parameters:
functionCovarianceFunction or PowerSpectralDensity

Model function associated to the Gaussian Process. Can be a covariance function or a power spectral density.

observation_indexesjax.Array

Indexes of the observed data, in this case it is the time.

observation_valuesjax.Array

Observation data.

observation_errorsjax.Array, optional

Errors on the observables, by default None

S_lowfloat, optional

Scaling factor to extend the frequency grid to lower values, by default 10. See PSDToACV for more details.

S_highfloat, optional

Scaling factor to extend the frequency grid to higher values, by default 10. See PSDToACV for more details.

methodstr, optional

Method to compute the covariance function from the power spectral density, by default ‘FFT’. Possible values are: - ‘FFT’: use the FFT to compute the autocovariance function. - ‘NuFFT’: use the non-uniform FFT to compute the autocovariance function. - ‘SHO’: approximate the power spectrum as a sum of SHO basis functions to compute the autocovariance function. - ‘DRWCelerite’: approximate the power spectrum as a sum of DRW+Celerite basis functions to compute the autocovariance function.

use_tinygpbool, optional

Use tinygp to compute the log marginal likelihood, by default False. Should only be used when the power spectrum model is expressed as a sum of quasi-separable kernels, i.e. method is not ‘FFT’ or ‘NuFFT’.

n_componentsint, optional

Number of components to use when using tinygp and the power spectrum model is expressed as a sum of quasi-separable kernels.

estimate_meanbool, optional

Estimate the mean of the observed data, by default True.

estimate_variancebool, optional

Estimate the amplitude of the autocovariance function, by default True.

scale_errorsbool, optional

Scale the errors on the observed data by adding a multiplicative factor, by default True.

log_transformbool, optional

Use a log transformation of the data, by default False. This is useful when the data is log-normal distributed. Only compatible with the method ‘FFT’ or ‘NuFFT’.

nb_prediction_pointsint, optional

Number of points to predict, by default 5 * length(observed(indexes)).

prediction_indexesjax.Array, optional

indexes of the prediction data, by default linspace(min(observation_indexes),max(observation_indexes),nb_prediction_points)

get_cov(xt: jax.Array, xp: jax.Array, errors: jax.Array | None = None) → jax.Array[source]#

Compute the covariance matrix between two arrays.

To compute the covariance matrix, this function calls the get_cov_matrix method of the model. If the errors are not None, then the covariance matrix is computed for the observationst, i.e. with observed data as input (xt=xp=observed data) and the errors on the measurement. The total covariance matrix is computed as:

\[C = K + \nu \sigma ^ 2 \times [I]\]

With \(I\) the identity matrix, \(K\) the covariance matrix, \(\sigma\) the errors and \(\nu\) a free parameter to scale the errors.

Parameters:
xt: :obj:`jax.Array`

First array.

xp: :obj:`jax.Array`

Second array.

errors: :obj:`jax.Array`, optional

Errors on the observed data

Returns:
jax.Array

Covariance matrix between the two arrays.

compute_log_marginal_likelihood_pioran() → float[source]#

Compute the log marginal likelihood of the Gaussian Process.

The log marginal likelihood is computed using algorithm (2.1) in Rasmussen and Williams (2006) Following the notation of the book, \(x\) are the observed indexes, x* is the predictive indexes, y the observations, k the covariance function, sigma the errors on the observations.

Solve of triangular system instead of inverting the matrix:

\(L = {\rm cholesky}( k(x,x) + \nu \sigma^2 \times [I] )\)

\(z = L^{-1} \times (\boldsymbol{y}-\mu))\)

\(\mathcal{L} = - \frac{1}{2} z^T z - \sum_i \log L_{ii} - \frac{n}{2} \log (2 \pi)\)

Returns:
float

Log marginal likelihood of the GP.
